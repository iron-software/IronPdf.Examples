## 1. Choosing Between CSS Media Type Settings
IronPDF offers two rendering modes: `Print` (default) and `Screen`, each with their advantages, as shown [at the end of this section](#anchor-comparison-of-screen-and-print-example).
- `CssMediaType.Print` optimizes HTML for physical printing, adjusting elements like backgrounds and icons for ink conservation—a standard choice for print previews without complex images.
- `CssMediaType.Screen` ensures the PDF looks exactly as it does on your computer screen, preserving all visual elements for web faithful rendering.

```cs
using IronPdf;
namespace ironpdf.PixelPerfectHtmlToPdf
{
    public class Section1
    {
        public void Run()
        {
            // Configure HTML formatting
            IronPdf.ChromePdfRenderer renderer = new IronPdf.ChromePdfRenderer();
            renderer.RenderingOptions.CssMediaType = PdfCssMediaType.Print; // or maybe Screen depending on the need
        }
    }
}
```
### Consistent Table Headers Across Pages
If your HTML includes a multi-page table, ensuring consistent header visibility is crucial.
- Headers reappear on each new page only when using `Print`.
- Conversely, `Screen` mode prints headers just once.

## 2. Configure Chrome for Alignment with IronPDF

Utilize IronPDF’s Chrome Rendering Engine effectively by making preliminary adjustments within Chrome.

Opt for the `Screen` CSS Media setting to ensure your PDFs contain all visual components usually skipped in print layouts.

### 2a. Utilizing Chrome’s DevTools to Emulate Media Settings

1. Open [Chrome's DevTools](https://developer.chrome.com/docs/devtools/open/).

   ![Open DevTools in Chrome](https://ironpdf.com/static-assets/pdf/how-to/pixel-perfect-html-to-pdf/open_devtools.gif)

2. Press Command+Shift+P (Mac) or Control+Shift+P (Other platforms) to access the **Command Menu**. Type 'rendering', select Show Rendering, and hit Enter to reveal the Rendering tab.

   ![Find and Set Emulate CSS Media in DevTools](https://ironpdf.com/static-assets/pdf/how-to/pixel-perfect-html-to-pdf/find_emulate_css_type.gif)

3. Scroll to the `Emulate CSS media` section and choose either `Screen` or `Print`. Reload the page (Ctrl+R) if necessary to apply settings.

### 2b. Activating the Print Preview in Chrome

Adjust your print settings to closely match your PDF’s intended look.

### 2c. Final Adjustments in Print Settings

- Select appropriate paper size (e.g., `A4` or `Letter`) as per your project.
- Select `Custom` from the `Margins` drop-down and set a uniform one-inch margin.
- Enable the `Background Graphics` to incorporate all visual elements.
- Choose `Landscape` layout if your document is designed horizontally.

Completing these steps prepares your HTML for precise debugging, reflecting how IronPDF will process the layout.

## 3. Configure IronPDF

### 3a. Matching IronPDF Settings with Chrome’s CSS Media Type
Ensuring alignment between the webpage design and the PDF output requires selecting the same CSS Media Type in your code as set in Chrome.

Remember, `PdfCssMediaType.Screen` includes comprehensive graphics, whereas `PdfCssMediaType.Print` focuses on ink conservation and may exclude certain visual aspects.

```cs
using IronPdf;
namespace ironpdf.PixelPerfectHtmlToPdf
{
    public class Section2
    {
        public void Run()
        {
            // Setup for rendering using Screen media type
            IronPdf.ChromePdfRenderer renderer = new IronPdf.ChromePdfRenderer();
            renderer.RenderingOptions.CssMediaType = PdfCssMediaType.Screen; // or switch to Print as needed
            renderer.RenderingOptions.PrintHtmlBackgrounds = true; // Ensure all background graphics are rendered
        }
    }
}
```
### 3b. Managing Rendering Delays and Timeouts
IronPDF defaults to a 60-second timeout for rendering. To prevent timeouts and ensure all content loads correctly, adjust the `RenderingOptions.Timeout`. Also, set a sufficient `RenderDelay` to allow elements like images and JavaScript to fully process.

Be cautious as setting no timeout or a render delay longer than your timeout may trigger a "**Unable to Render PDF**" error. Adjustments may be necessary based on your specific content needs.

```cs
using IronPdf;
namespace ironpdf.PixelPerfectHtmlToPdf
{
    public class Section3
    {
        public void Run()
        {
            // Adjust Timeout and RenderDelay for optimal rendering
            renderer.RenderingOptions.Timeout = 90; // Extend if necessary beyond the default 60 seconds
            renderer.RenderingOptions.WaitFor.RenderDelay(30000); // Delay adjusted for content loading
        }
    }
}
```
